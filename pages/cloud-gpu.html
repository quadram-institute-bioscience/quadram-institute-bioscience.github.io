What is the L40S?

A GPU built for science

Most people are familiar with consumer gaming GPUs. The L40S is in a completely different class: a professional, data-centre-grade card that NVIDIA sells to research labs, universities, and cloud providers for AI and HPC workloads.

Put simply…

  • 🎮
    RTX 4090 — the fastest consumer GPU you can buy for gaming. It has 24 GB VRAM. It can run a 7B language model. The L40S has twice the VRAM and more raw compute.
  • 💻
    A typical workstation GPU (RTX 3080) has 10 GB VRAM. You can barely load a 7B model in 4-bit on it. The L40S has nearly 5× the VRAM.
  • ☁️
    Entry-level cloud GPU (T4) — commonly used for inference, 16 GB VRAM, ~8 TFLOPS. The L40S delivers 11× the compute and 3× the memory.
  • 🔬
    For bioinformatics: protein folding (AlphaFold2/3), genomic foundation models, single-cell embeddings, sequence-to-function prediction — all fit comfortably within this GPU's memory envelope.
One sentence: The L40S gives Quadram researchers access to the same GPU class used by AI labs without a cloud bill measured in thousands of pounds per week.

VRAM: the resource that matters most

VRAM (GPU memory) is the hard limit on what you can run. More VRAM = larger models, larger batches, and faster throughput without offloading to system RAM.

Getting Started

Set up your GPU environment

Follow these steps once to have a fully working GPU Python environment. All commands run as your regular user — no root required after the reboot.

1

Add CUDA to your PATH

The CUDA toolkit is installed at /usr/local/cuda but its bin/ directory is not on the default $PATH. Add these lines to ~/.bashrc:

~/.bashrc
export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
Reload & verify
source ~/.bashrc
nvcc --version   # should print: release 13.2, V13.2.78
2

Create a GPU micromamba environment

Create a dedicated environment called gpu with PyTorch and the CUDA 12.4 runtime. Even though the system CUDA is 13.2, PyTorch bundles its own CUDA runtime — CUDA 12.4 builds are the current stable target and work perfectly on the L40S.

bash
micromamba create -n gpu \
  -c pytorch -c nvidia -c conda-forge \
  python=3.11 \
  pytorch torchvision torchaudio pytorch-cuda=12.4 \
  jupyter ipykernel numpy pandas matplotlib scipy \
  -y
3

Verify GPU access (after reboot)

Run the following to confirm PyTorch can see the L40S:

bash
micromamba run -n gpu python -c "
import torch
print('PyTorch :', torch.__version__)
print('CUDA OK :', torch.cuda.is_available())
print('GPU     :', torch.cuda.get_device_name(0))
print('VRAM    :', round(torch.cuda.get_device_properties(0).total_memory / 1e9, 1), 'GB')
"

Expected output:

expected output
PyTorch : 2.x.x
CUDA OK : True
GPU     : NVIDIA L40S
VRAM    : 47.4 GB
4

Register a Jupyter kernel (optional)

Make the environment available as a Jupyter kernel:

bash
micromamba run -n gpu \
  python -m ipykernel install --user \
  --name gpu --display-name "GPU (PyTorch / CUDA)"
Advanced Usage

Making the most of 48 GB VRAM

Recipes for LLM inference, fine-tuning, FP8 precision, and the HuggingFace ecosystem.

LLM Inference with vLLM

vLLM is the fastest open-source LLM serving engine. It uses paged attention and continuous batching to fully utilise the 48 GB VRAM.

Install
micromamba activate gpu
pip install vllm
Serve a 13B model via API
vllm serve meta-llama/Llama-3.1-8B-Instruct \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.90
Run a 70B model in 4-bit
vllm serve Qwen/Qwen2.5-72B-Instruct \
  --quantization awq \
  --max-model-len 4096

Fine-tuning with PEFT / LoRA

Parameter-efficient fine-tuning (LoRA/QLoRA) lets you adapt large models to domain-specific tasks (food science, genomics, clinical text) with a fraction of the compute cost.

Install HuggingFace stack
pip install transformers accelerate \
  datasets peft bitsandbytes trl
Minimal QLoRA fine-tune (Python)
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
from peft import get_peft_model, LoraConfig
import torch

bnb_cfg = BitsAndBytesConfig(load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.bfloat16)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3.1-8B",
    quantization_config=bnb_cfg,
    device_map="cuda")

lora_cfg = LoraConfig(r=16, lora_alpha=32,
    target_modules=["q_proj","v_proj"])
model = get_peft_model(model, lora_cfg)
model.print_trainable_parameters()
# trainable params: ~4M (0.05% of total)

FP8 Training (Ada Lovelace native)

The L40S supports hardware-accelerated FP8 via the Transformer Engine library. This gives 2–3× speedup over FP16 and uses roughly half the VRAM.

Install
pip install transformer-engine[pytorch]
Use FP8 in a training loop
import transformer_engine.pytorch as te
from transformer_engine.common.recipe import (
    Format, DelayedScaling)

fp8_recipe = DelayedScaling(
    fp8_format=Format.HYBRID,
    amax_history_len=16,
    amax_compute_algo="max")

with te.fp8_autocast(enabled=True, fp8_recipe=fp8_recipe):
    output = model(input_ids)

import esm, torch

model, alphabet = esm.pretrained.esm2_t36_3B_UR50D()
model = model.cuda().half()

batch_converter = alphabet.get_batch_converter()
data = [("protein1", "MKTIIALSYIFCLVFA")]
_, _, tokens = batch_converter(data)

with torch.no_grad():
    out = model(tokens.cuda(), repr_layers=[36])
